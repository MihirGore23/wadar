#define DEMO_TITLE   "Cayenne - Clutter Demo"

Desktop ClutterDemo

This is a clutter map demonstration which works in conjunction with the Cayenne RadarServer app. Experiment with each type of clutter map to get a sense of how each works.

Note
Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

In this demo, the RadarServer app on the Cayenne is used to generate the radar data which is transferred using TCP/IP sockets. It uses the SalsaComm protocol as defined in the common demo folder.

Note
This demo runs on a Linux desktop, which is likely many times faster than a BeagleBone.

Hardware Setup

For this demo a BeagleBone Black + an Cayenne cape + antenna cape was connected to Linux desktop PC using the USB cable.

On the BBB, start the radar server using the command: 

# ./radarServer 11235

The command-line argument is the port number to use. It can be any open port; just make sure to also use this same argument with the client.

Demonstration Code

Start the demo using the command: 

# ./clutterDemo [server ip] [server port]

When using the USB cable, the [server ip] is 192.168.7.2. The [server port] should match whatever port was used when starting up the radar server app.

Once connected, a GUI window will appear. Initially, the screen will be all Black. There is a prompt on the bottom. To start sending radar data, hit the [spacebar] key.

Initial Screen

Once the server starts sending radar frames, the window will update. The radar data is represented in a few ways. The RED line represents the processed radar signal vs time. In this demo, the processed signal is the radar signal with the clutter removed. Additionally, behind this RED line is the B-Scan view, where shades of Grey are used to represent signal amplitude. This representation displays a history. New signals appear at the top, and older signals are pushed down. Amplitudes which are small will appear white, while larger amplitudes are darker.

The X1 radar used in the Cayenne has an ~ 2 m frame width without stitching. In the image below, notice the stationary second reflection; this is the ceiling. The other reflection was for a small metal plate which was moved up/down over the radar.

Command-line Options

SalsaViewer keyboard options:

  • [ ESC ] - quits the program
  • [space] - toggles pause
  • [ 1 ] - toggle A-Scan on/off
  • [ 2 ] - toggle B-Scan on/off
  • [ R ] - resets vertical scale

Additional keyboard commands supported:

  • [ C ] - Initiate radar timing calibration (code is part of radarClient.c)
  • [ A ] - Enable adaptive clutter map
  • [ S ] - Enable static clutter map
  • [ 3 ] - Enable 3-pulse MTI clutter map
  • [ 5 ] - Enable 5-pulse MTI clutter map

Additional keyboard commands related to adaptive filter

  • [ UP ] - Increase adaptive alpha
  • [ DOWN ] - Decrease adaptive alpha
  • [ RIGHT ] - Increase adaptive beta
  • [ LEFT ] - Decrease adaptive beta
Note
The clutter maps are mutually exclusive.
It is possible that depending on the target position for the actual amplitude to exceed the vertical scale. Press the 'R' key to rescale at any time
On the Cayenne, the radar timing measurement, or "calibration" takes just a few seconds to complete.. During this time, no radar signals are sent since the radar is busy. If it had been sending radar frames, it should resume once the calibration is complete.
On Oracle VirtualBox when the demo is run, the console will display the lines: 
libGL error: pci id for fd 5: 80ee:beef, driver (null)
libGL error: core dri or dri2 extension not found
libGL error: failed to load driver: vboxvideo
This seems to be normal behavior.
