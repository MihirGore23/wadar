Desktop MotionDetect

This is a simple motion detection demonstration which works in conjunction with the Cayenne RadarServer app.

Note
Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

In this demo, the RadarServer app on the Cayenne is used to generate the radar data which is transferred using TCP/IP sockets. It uses the SalsaComm protocol as defined in the common demo folder.

Note
This demo runs on a Linux desktop, which is likely many times faster than a BeagleBone.

Hardware Setup

For this demo a BeagleBone Black + an Cayenne cape + antenna cape was connected to Linux desktop PC using the USB cable.

On the BBB, start the radar server using the command: 

# ./radarServer 11235

The command-line argument is the port number to use. It can be any open port; just make sure to also use this same argument with the client.

Demonstration Code

Start the demo using the command: 

# ./motionDetect [server ip] [server port]

When using the USB cable, the [server ip] is 192.168.7.2. The [server port] should match whatever port was used when starting up the radar server app.

Once connected, a GUI window will appear. Initially, the screen will be all Black. There is a prompt on the bottom. To start sending radar data, hit the [spacebar] key.

Initial Screen

Once the server starts sending radar frames, the window will update. The radar data is represented in a few ways. The RED line represents distance vs. time of the radar signal in the time domain with the clutter removed. In general this line will be flat and centered in the screen. The far left represents the distance where the frame begins. The distance increases along the horizontal axis. Additionally, behind this RED line is the B-Scan view, where shades of Grey are used represent signal amplitude. This representation displays a history. New signals appear at the top, and older signals are pushed down.

The DSP in this demo is very simple. The signal is examined using a forward search. If any amplitude is beyond a threshold, then a distance is computed and a Green mark is made at that point on the B-Scan view and the distance is reported on the window. The user can adjust the threshold % using the up/ down arrow keys.

The % is relative to the default vertical scale. The top of the window is YMAX and the bottom is -YMAX. The threshold is the % times YMAX.

The demo concept is to show how the MTI filter with a threshold comparison can be used for movement detection.

Command-line Options

SalsaViewer keyboard options:

  • [ ESC ] - quits the program
  • [space] - toggles pause
  • [ 1 ] - toggle A-Scan on/off
  • [ 2 ] - toggle B-Scan on/off
  • [ R ] - resets vertical scale

There are several additional keyboard commands supported:

  • [ C ] - Triggers radar timing calibration
  • [ UP ] - Increase movement threshold %
  • [DOWN] - Decrease movement threshold %
  • [ V ] - Reset vertical scaling to default
Note
Since the app runs fast, when one of the indicated keys is pressed, the event may trigger multiple times. In the case of calibration, an enable boolean is used to prevent this. However, for the others, nothing extra is done.

Example Output

The Cayenne was mounted to a tripod, placed on the edge of a desk, where the antennas were pointed horizontally parallel to the floor. A human target walked diagonally away and towards the radar a few times.

Demo in action
Note
On the Cayenne, the radar timing measurement, or "calibration" takes just a few seconds to complete. During this time, no radar signals are sent since the radar is busy. If it had been sending radar frames, it should resume once the calibration is complete.

libGL error: core dri or dri2 extension not found
libGL error: failed to load driver: vboxvideo
This seems to be normal behavior.
