RadarServer

Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

This demo is very simple TCP server which enables a remote TCP client to quickly stream radar data at a much faster framerate than is possible with Gnuplot.

The server is started from the command-line: 

# ./radarServer 11111

The command-line argument is the port number to use.

Example session: 

# ./radarServer 11111
Calibrating radar (may take several seconds)
Server ready...
Handling client 192.168.7.1

When the server is started it will open a handle to the radar and initiate a radar timing measurement, or calibration. Once the calibration is finished, the server indicates it's ready for a client to connect.

The user can hit [CTRL-C] to stop the server and quit the program.

Client ⇒ Server protocol:

<header><data> = 8 bytes total

where:
<header> = <cmd><len>

The <cmd> is a uint16_t type. The cmd codes are defined in salsaComm.h. The <len> is a uint16_t type and is fixed to 4 for this demo. The <data> is also 4 bytes, but for this implementation the payload is not examined.

In the Radarlib3 API, settings are either int or float values, so in some future version, the <data> portion could be used for dynamic settings changes, such as the OffsetDistanceFromReference.

Server ⇒ Client protocol:

<header><data> = N + 4 bytes total

where:
<header> = <cmd><len> = 4 bytes
<data> = N bytes

The <data> is context sensitive based on what data is being transmitted. In general, this is radar counter data which is composed of arrays of uint32_t values. The number of which is dependent on radar settings. In some cases, the <data> portion will not be sent (if N == 0).

The asymmetry between the client/server interaction is based on using the socket option to set a read timeout in the client handling function. In the client handler function, once the client indicates data should be sent, it is then done so repeatedly without the client requesting it.
