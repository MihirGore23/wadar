ClutterDemo

This is a simple program which demonstrates the use of the clutter map API with the Onza and a Cilantro Switching cape.

Hardware Setup

For this demo, a BeagleBone Black + Onza cape + LTSA 5 antennas were used along with a Cilantro Switching Cape. The orientation doesn't matter.

Demonstration Code

When a handle to the fex4 driver is opened, there is some immediate feedback; the X4 initialization performs several checks on startup. Typically, the INFO statements will happen quickly. On any failure, there are some retries which delay the output somewhat.

There is also some output shown about the X4 configuration.

Demo in Action

Once the radar has been configured, a loop begins where the radar signal with clutter removed is continually plotted via Gnuplot. In order to see the plot, make sure that the -X option is used when making the SSH connection to the BeagleBone Black.

For example: 

ssh -X root@192.168.7.2

There are several types of clutter maps used:

  • static clutter map
  • adaptive clutter map
  • MTI clutter map

When using the static clutter map (the default), the first radar signal is used to set the static clutter map. This can work well if no targets are in view when the program starts. As the program runs, this static clutter is removed from each subsequent signal.

If the -a option is used, then either the adaptive clutter map or the MTI clutter map is enabled.

When using the adaptive clutter map, a portion of each new signal is added to the clutter map, and a portion is removed during the subtraction. When using the adaptive clutter map, a target at a fixed position will adapt out over time. The user can adjust the adaptive alpha and beta factors to see the effect on the signal. A larger "alpha" factor will remove a larger % of the clutter map from the signal. A larger "beta" will result in shorter history. Both the alpha and beta parameters are adjustable via the -p and -b command-line options.

MTI stands for moving target indicator; it works like a faster adaptive clutter map. There are two options, a 3-pulse and a 5-pulse type. The MTI clutter map is appropriate in situations for detecting motion. It will tend to "adapt" out more quickly than the adaptive clutter map.

Example - Static Clutter Map

In this example, the radar was placed on the edge of a table with the antennas oriented horizontally. The initial radar signal captured was used to set the static cluttermap. Next, a small metal plate was held in front of the radar ~ 385 cm away; its reflection is the large amplitude pulse just past sample 600. There is still some near-field clutter present below sample 200.

Example - Adaptive Clutter Map

In this example, the "-a 0" command-line option was used to enable the adaptive clutter map. Once again, the metal plate was held in front of the radar. If the plate is held still, it tends to adapt out. This time, with the target at a similar distance, the near-field clutter was adapted out almost completely.
