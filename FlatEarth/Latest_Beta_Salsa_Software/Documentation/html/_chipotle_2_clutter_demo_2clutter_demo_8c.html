ClutterDemo

This is a simple program which demonstrates the use of the clutter map API; this program is based on the RadarDemo project.

Hardware Setup

For this demo, a BeagleBone Black + a Chipotle cape + 5" antennas and AVA 0.8 were used. The orientation doesn't matter.

Demonstration Code

Once a handle to the radar is opened, the timing circuits can be calibrated. This is optional but suggested. Using the same settings, run the program with and without the -c option to see the effect. There should be a difference in the sampling rate, the sample resolution, and the signal will appear to be shifted.

Examples

In this first run, the timing measurement was not executed.

Console output: 

OffsetDistanceFromReference = 0.000000 m
Sample Delay = 3.340000 ns
SamplesPerSecond = 38000001024
Sample resolution = 0.0039 m
NumSamplers = 512
Starting radar loop... (hit ESC key to quit)

In the second run, this time, the -c command-line option was used to execute the radar timing measurement.

Console output: 

Running radar timing calibration...
OffsetDistanceFromReference = 0.000000 m
Sample Delay = 3.340000 ns
SamplesPerSecond = 35649695744
Sample resolution = 0.0042 m
NumSamplers = 512
Starting radar loop... (hit ESC key to quit)

Notice that the SamplesPerSecond is different. When the timing measurement is made, this value is measured; SamplesPerSecond is used to compute the sample resolution, which is used for distance estimation. The timing measurement is also especially important when frame stitching is used. Additionally, on the X1, if the SamplingRate variable was set to 1, then the hard-coded sampling rate would be off by at least a factor of 2!

Radar Configuration

The project directory contains some files with the extension ".json". These files are used to alter the radar configuration. A few specific X1 variables are highlighted below.

The 'ClkDivider' setting controls the radar's pulse repetition frequency (PRF). This is set to 1, which sets the PRF to 50 MHz. This value is actually default, but is set to explicitly indicate the desired PRF.

The 'SamplingRate' setting controls the spatial resolution of the radar frame bins. The indicated setting should produce a resolution of ~ 8mm. If the timing measurements are not executed (the default program behavior) the reported resolution will display as ~ 4mm – an incorrect value.

Command-line Options

There are several command-line options:

  • -a : enable adaptive/MTI clutter map
  • -c : execute the radar timing measurement
  • -o : save the JSON configuration to a file
  • -p : adjust adaptation "alpha" factor (for adaptive clutter map)
  • -b : adjust adaptation "beta" factor (for adaptive clutter map)

Demo in Action

Once the radar has been configured, a loop begins where the radar signal with clutter removed is continually plotted via Gnuplot. In order to see the plot, make sure that the -X option is used when making the SSH connection to the BeagleBone Black.

For example: 

ssh -X root@192.168.7.2

There are several types of clutter maps used:

  • static clutter map
  • adaptive clutter map
  • MTI clutter map

When using the static clutter map (the default), the first radar signal is used to set the static clutter map. This can work well if no targets are in view when the program starts. As the program runs, this static clutter is removed from each subsequent signal.

If the -a option is used, then either the adaptive clutter map or the MTI clutter map is enabled.

When using the adaptive clutter map, a portion of each new signal is added to the clutter map, and a portion is removed during the subtraction. When using the adaptive clutter map, a target at a fixed position will adapt out over time. The user can adjust the adaptive alpha and beta factors to see the effect on the signal. A larger "alpha" factor will remove a larger % of the clutter map from the signal. A larger "beta" will result in shorter history. Both the alpha and beta parameters are adjustable via the -p and -b command-line options.

MTI stands for moving target indicator; it works like a faster adaptive clutter map. There are two options, a 3-pulse and a 5-pulse type. The MTI clutter map is appropriate in situations for detecting motion. It will tend to "adapt" out more quickly than the adaptive clutter map.

Example - Static Clutter Map

In this example, the radar was placed on a table with the antennas facing out horizontally, parallel with the floor. The initial radar signal captured was used to set the static clutter map. Next, a small metal plate was held in front of the radar; its reflection is the large amplitude pulse around sample 450.

Static Clutter Removed

Example - Adaptive Clutter Map

In this example, the "-a 0" command-line option was used to enable the adaptive clutter map. Once again, the metal plate was in front of the radar. With the adaptive clutter map, things like the near-field are constantly adapted out so there are no additional reflections. If the plate is held still, it also would disappear. Additionally, the "-p 1.0" option was used to set the alpha parameter. This was to make it easier to capture an image.
