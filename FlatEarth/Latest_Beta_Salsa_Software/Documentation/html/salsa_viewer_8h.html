SalsaViewer

The Salsa Viewer is a simple API for visualizing data. It is intended to be used with the Salsa radar devices, so it is a tool to help view radar signals. It can be used on a desktop PC and should operate much faster than is possible with Gnuplot using piping.

The salsaViewer uses the SFML API for its graphics. SFML is a lightweight, multi-platform, multi-language API.

To install 'C' version of SFML on an Ubuntu system: 

sudo apt-get install libcsfml-dev

Make sure to add the following to the linking step in the makefile 

-lcsfml-graphics -lcsfml-window -lcsfml-system

Before using the viewer, it must be created 

SalsaViewer viewer;

// Create a viewer, 640 x 480 pixels where each signal is 512 samples, with a signal history of 200 signals
viewer = SalsaViewer_create(640, 480, 512, 200, "../Common/resources/DejaVuSansMono.ttf");

The bottom portion of the window provides an area to display text. Text will display faster here than with stdout on the console. The user's app should provide storage for the string.

char userText[80];
...
// Update the string
snprintf(userText, 80, "Press [spacebar] to start");

// Update the text in the viewer window
SalsaViewer_updateText(viewer, userText);

On creation, the viewer is internally in a paused state. This has no effect on the use of most functions, but is provided just as a convenience for the user. Since this viewer runs on a desktop, a typical use might be to "play back" data collected previously for DSP development/experimentation.

Typical program flow: 

...
while (SalsaViewer_isOpen(viewer))
{
  // Get user input
  SalsaViewer_getInput(viewer);

  if (!SalsaViewer_isPaused(viewer)) {
    // code to get radar data
    // code to process radar data

    // code to update user text

    // Update the viewer
    SalsaViewer_updateData(viewer, radarSig);
    SalsaViewer_updateText(viewer, userText);
  }

  // Display the latest view
  SalsaViewer_display(viewer);
}
...

When the viewer is created, it assumes maximum vertical amplitude is 1.0. To reset the vertical scale, the user can hit the [R] key (at any time) or call the SalsaViewer_resetVScale() function to reset the vertical scale.

The SalsaViewer_getInput() provides support for several keys:

  • [ ESC ] - quits the program
  • [space] - toggles pause
  • [ 1 ] - toggle A-Scan on/off
  • [ 2 ] - toggle B-Scan on/off
  • [ R ] - resets vertical scale
