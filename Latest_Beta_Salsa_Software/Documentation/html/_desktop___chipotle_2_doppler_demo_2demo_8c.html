Desktop DopplerDemo

This is a Doppler demonstration which works in conjunction with the Chipotle RadarServer app. The main purpose of this version is to help visualize the Doppler matrix data.

Note
Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

In this demo, the RadarServer app on the Chipotle is used to generate the radar data which is transferred using TCP/IP sockets. It uses the SalsaComm protocol as defined in the common demo folder.

Note
This demo runs on a Linux desktop, which is likely many times faster than a BeagleBone.

Hardware Setup

For this demo a BeagleBone Black + an Chipotle cape + antenna cape was connected to Linux desktop PC using the USB cable.

On the BBB, start the radar server using the command: 

# ./radarServer 11235

The command-line argument is the port number to use. It can be any open port; just make sure to also use this same argument with the client.

Demonstration Code

Start the demo using the command: 

# ./dopplerDemo [server ip] [server port]

When using the USB cable, the [server ip] is 192.168.7.2. The [server port] should match whatever port was used when starting up the radar server app.

Once connected, a GUI window will appear. Initially, the screen will be all Black. There is a prompt on the bottom. To start sending radar data, hit the [spacebar] key.

Initial Screen

Once the server starts sending radar frames, the window will update. The radar data is represented in a few ways. Initially, the display may look weird. Depending on the number of Doppler columns, it takes some time for the Doppler matrix to converge. Once the Doppler matrix has converged, most of the screen area should be Grey. The horizontal axis represents velocity, with the center equal to 0 velocity. The vertical axis is distance. The top of the window represents the start of the radar frame and distances increase downward.

When a target is within view, the area around the target will look like blobs of white/black. It will appear on one side or depending on the direction of travel. The bottom of the window will display some text which may indicate distance to target and direction of travel. There is an adjustable threshold to make the app more or less sensitive.

Note
As the demo operates, it is possible the plot will show some aliasing of the data. When the target approaches - there may also be "blobs" moving the other direction. This may be real, but this may also be an aliasing effect based on the framerate. For less aliasing, modify the radar server settings to make faster radar measurements, such as increasing the DACStep value...

Command-line Options

SalsaViewer keyboard options:

  • [ ESC ] - quits the program
  • [space] - toggles pause

Additional keyboard commands supported:

  • [ UP Arrow ] - Increase threshold
  • [DOWN Arrow] - Decrease threshold
