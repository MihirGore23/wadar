Desktop RangingDemo

This is a radar ranging demonstration which works in conjunction with the Chipotle radarServer app.

Note
Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

In this demo, the RadarServer app on the Chipotle is used to generate the radar data which is transferred using TCP/IP sockets. It uses the SalsaComm protocol as defined in the common demo folder.

Note
This demo runs on a Linux desktop, which is likely many times faster than a BeagleBone.

Hardware Setup

For this demo a BeagleBone Black + an Chipotle cape + antenna cape was connected to Linux desktop PC using the USB cable.

On the BBB, start the radar server using the command: 

# ./radarServer 11235

The command-line argument is the port number to use. It can be any open port; just make sure to also use this same argument with the client.

Demonstration Code

Start the demo using the command: 

# ./rangingDemo [server ip] [server port]

When using the USB cable, the [server ip] is 192.168.7.2. The [server port] should match whatever port was used when starting up the radar server app.

Once connected, a GUI window will appear. Initially, the screen will be all Black. There is a prompt on the bottom. To start sending radar data, hit the [spacebar] key.

Initial Screen

Once the server starts sending radar frames, the window will update. The radar data is represented in a few ways. A number of steps are made to convert the radar counter data into the processed signal shown. First, the clutter is removed. Next, the mean is removed, and envelope of the result is computed. This envelope is finally run through a smoothing filter. The far left represents the distance where the frame begins. The distance increases along the horizontal axis. Additionally, behind the envelope is the B-Scan view, where shades of Grey are used to represent signal amplitude. This representation displays a history. New signals appear at the top, and older signals are pushed down. Amplitudes which are small will appear white, while larger amplitudes are darker. When the amplitude is greater than a threshold, there will also be a Cyan tracking line shown on the B-Scan; this represents first peak greater than the threshold.

Each time the radar signal is updated, the peak amplitude of the envelope is computed. If this value is greater than a threshold, the distance to the peak is computed.

Command-line Options

SalsaViewer keyboard options:

  • [ ESC ] - quits the program
  • [space] - toggles pause
  • [ 1 ] - toggle A-Scan on/off
  • [ 2 ] - toggle B-Scan on/off
  • [ R ] - resets vertical scale

There are several additional keyboard commands supported:

  • [ C ] - Initiate radar timing calibration (code is part of radarClient.c)
  • [ A ] - Enable adaptive clutter map
  • [ S ] - Enable static clutter map
  • [ V ] - Reset vertical scale back to default
Note
The static and adaptive clutter maps are mutually exclusive. Using one will disable the other.

Example Output

The Chipotle was mounted to a tripod, placed on the edge of a desk, where the antennas were pointed horizontally parallel to the floor. A human target walked towards and away from radar a few times.

In the image shown, in much of the B-Scan, the maximum amplitude of the envelope was below the threshold, so the Cyan tracking didn't show. However, the B-Scan clearly shows changes in amplitude throughout the frame. This is where specific tuning could be done to detect and ignore targets.

Demo in action
Note
It is possible that depending on the target position for the actual amplitude to exceed the vertical scale. In this situation, the amplitude values shown are clipped to a maximum value depending on the predefined vertical scale.
The SalsaViewer itself has a number of keyboard key options. 'R' resets the vertical scale based on the current signal. Hitting 'R' can cause the B-Scan view to look odd. To reset the scale back to normal, the user can press the 'V' key.
On the Chipotle, the radar timing measurement, or "calibration" takes just a few seconds to complete.. During this time, no radar signals are sent since the radar is busy. If it had been sending radar frames, it should resume once the calibration is complete.
On Oracle VirtualBox when the demo is run, the console will display the lines: 
libGL error: pci id for fd 5: 80ee:beef, driver (null)
libGL error: core dri or dri2 extension not found
libGL error: failed to load driver: vboxvideo
This seems to be normal behavior.
