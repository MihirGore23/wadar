DDC Ranging Demo

This is a simple program which makes use of the dsp_helper api, specifically the digital downconversion (DDC) which is an alternative way to compute the radar signal envelope.

In this version, no cluttermaps are used, so the data is using the radar "raw" signal.

Note
The dsp_helper api includes some useful DSP code which is not present in the salsalib api. The dsp_helper api provides a software DDC function for the X4.

Hardware Setup

For this demo, a BeagleBone Black + Onza cape + LTSA 5 antennas were used. The orientation doesn't matter.

Demo in Action

The radar was placed on a table with the antennas facing horizontally. A metal plate was attached to the front of a cart which was moved toward and away from the radar.

The line:

status |= fex4_setSampleDelay(x4, 2); // the direct-path will be out-of-frame

was uncommented so the direct-path reflection between the antennas was not in frame; if it is commented-out, then the direct-path would be the "max" peak so the distance estimate would be ~ 0 m.

When the app is running, two plots are shown. The red plot represents the "raw" radar signal normalized into volts. The blue represents the signal envelope which is computed using a digital downconversion (DDC) technique.

Signal Envelope

User Enhancements

In this version, no target thresholds are used. The zone sense concept is also not used. This was just a demo of the DDC technique in action.
