Desktop Jalapeno Radar Demo

This is a simple demonstration which works in conjunction with the Jalapeno jalapenoServer demo.

Note
Many of the other demo programs utilize Gnuplot with X11 forwarding for radar signal visualization. In this scenario, Gnuplot is actually running on the BeagleBone platform. This limits the radar's framerate.

In this demo, the jalapenoServer demo on the Jalapeno is used to generate the radar data which is transferred using TCP/IP sockets. It uses the JalapenoComm protocol as defined in the common demo folder.

Note
This demo runs on a Linux desktop, which is likely many times faster than a BeagleBone.

Hardware Setup

For this demo a BeagleBone Black + an Jalapeno cape was connected to Linux desktop PC using the USB cable.

On the BBB, start the radar server using the command: 

# ./jalapenoServer 11235

The command-line argument is the port number to use. It can be any open port; just make sure to also use this same argument with the client.

Demonstration Code

Start the demo using the command: 

# ./rawDataDemo [server ip] [server port]

When using the USB cable, the [server ip] is 192.168.7.2. The [server port] should match whatever port was used when starting up the radar server app.

Once connected, a GUI window will appear. Initially, the screen will be all Black. There is a prompt on the bottom. To start sending radar data, hit the [spacebar] key.

Command-line Options

SalsaViewer keyboard options:

  • [ ESC ] - quits the program
  • [space] - toggles pause
  • [ 1 ] - toggle A-Scan on/off
  • [ 2 ] - toggle B-Scan on/off
  • [ R ] - resets vertical scale

Additional keyboard commands supported:

  • [ F ] - Toggle FFT on/off
  • [ E ] - Toggle EQ on/off
  • [ Y ] - Toggle RAMP on/off
  • [ K ] - Decrease LNA gain
  • [ L ] - Increase LNA gain
  • [ O ] - Decrease PGA gain
  • [ P ] - Increase PGA gain
  • [ Z ] - Select Channel 1
  • [ X ] - Select Channel 2
  • [ C ] - Select Channel 3
  • [ V ] - Select Channel 4
Note
The LNA/PGA/EQ settings effect the currently selected channel. Channel 1 is the default channel. The user can switch channels using the keyboard commands; there is no feedback directly from the server on the BBB, but there should be a visual difference on the screen.
